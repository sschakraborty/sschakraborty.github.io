The Gradient Descent Algorithm

Introduction

Gradient descent is an optimization algorithm that helps us find a global minima of a function.

It works by starting from a random point on the surface of the function and taking little (baby) steps of progression towards the steepest downward gradient.

Standing at any point, the next point to move to is decided by the direction of steepest downward gradient (or steepest gradient descent).

GD example and learning rate

Let us consider a function f(x)=(x−20)2f(x) = (x-20)^2f(x)=(x−20)2

The gradient of the function w.r.t xxx is

∂f(x)∂x=fx(x)=2(x−20)\frac{\partial{f(x)}}{\partial{x}}=f_{x}(x)=2(x-20)∂x∂f(x)​=fx​(x)=2(x−20)

So considering the current surface of the function we may start from a random point and take little steps towards the direction of steepest descent.

The amount of progress we make towards that direction (in one iteration) is called "Learning rate"

Learning rate

The Learning rate (denoted by α\alphaα) is a hand-picked contant that is used to depict the amount of progress the GD algorithm makes in the direction of steepest descent.

  • A small value for learning rate can make the algorithm slow. Gradient descent will take a long time to converge.
  • A large value of learning rate may completely miss the point of global minima and enter into an infinite loop.

Simple example (contd.)

So the algorithm goes as follows:

Repeat until convergence {
            x(n+1):=x(n)−α∂f(x)∂x∣x=x(n)x^{(n+1)} := x^{(n)} - \alpha \frac{\partial{f(x)}}{\partial{x}}|_{x=x^{(n)}}x(n+1):=x(n)−α∂x∂f(x)​∣x=x(n)​
}

Condition for convergence is

if | X_old - X_new | <= threshold {
	stop and return X_new
} else {
	X_old := X_new
}

Simple example (Implementation)

import random as rand

def df(x):
    return 2 * (x - 20)

def grad_descent(alpha, t):
    x_old = rand.random()
    while True:
        x_new = x_old - alpha * df(x_old)
        if abs(x_new - x_old) <= t:
            return x_new
        else:
            x_old = x_new

print(grad_descent(0.1, 0.00001))
# Output: 19.99996328116975

∂f(x,y)∂y=fy(x,y)=2(x+y2−20)(2y)\frac{\partial{f(x,y)}}{\partial{y}}=f_y(x,y)=2(x+y^2-20)(2y)∂y∂f(x,y)​=fy​(x,y)=2(x+y2−20)(2y)

Let (xmin,ymin)(x_{min}, y_{min})(xmin​,ymin​) be the point of global minima.

Using Gradient Descent

The algorithm would work as follows

Repeat until convergence {
            x(n+1):=x(n)−α∂f(x,y)∂x∣x=x(n)x^{(n+1)} := x^{(n)} - \alpha \frac{\partial{f(x,y)}}{\partial{x}}|_{x=x^{(n)}}x(n+1):=x(n)−α∂x∂f(x,y)​∣x=x(n)​

Generalized Gradient Descent

A generalized version of Gradient Descent algorithm for k-variable function

Defining a function

Let us consider a function as follows:
f(x1,x2,x3,...,xk)f(x_1, x_2, x_3,..., x_k)f(x1​,x2​,x3​,...,xk​)

Here x1,...,xkx_1,...,x_kx1​,...,xk​ are the kkk parameters that function fff depends on.

We assume the fact that function fff is always concave or atleast it has a well defined global minima to which Gradient Descent can converge.

Note: GD may converge to a local minima.

Vectorizing Gradient Descent

Gradient Descent algorithm can be vectorized. Vectorization allows us speeding up calculations as it mainly involves manipulation of matrices some basic Linear Algebra operations.

Linear Algebra operations can work on many independent pieces of data at the same time and parallelize the operations on them. This feature can be exploited and matrix operations can be sped up using GPUs and SIMD instruction sets.

Some conventions of Linear Algebra

Let AAA and BBB be two matrices. Matrix multiplications can be represented as A.BA.BA.B or simply ABABAB.

On the other hand, there is another type of multiplication called Hadamard Product, which simply means element wise multiplication.

Hadamard Product is represented as A⊙BA \odot BA⊙B

Disadvantages

  • Sometimes calculating the gradient can be very expensive or intractable if the size of your data is large.

  • Knowing how far to travel along the gradient can be a big problem, especially when you have a very non-convex objective function. This can be addressed through line-search methods (Armijo-Wolfe conditions) which pick appropriate step sizes.

Types of Gradient Descent

  • Batch Gradient Descent
    • Considers the whole training dataset per iteration.
  • Mini-batch Gradient Descent
    • Considers a randomly shuffled finite subset of the training dataset per iteration.
  • Stochastic Gradient Descent
    • Considers a single training item per iteration to calculate gradient.
